# 访问日志插件使用教程

> 模块路径：`utils/模块文件数据库/access_logger.py`

## 概述

访问日志插件用于记录和可视化所有HTTP请求信息。支持Django中间件自动记录、终端彩色输出、按大小自动切割日志、密码保护的Web可视化页面和图表统计。

**特点**：
- ✅ 自动记录所有请求：路径、方法、IP、UA、状态码、耗时、域名
- ✅ 文本文件存储，按大小自动切割（GZIP压缩备份）
- ✅ 终端彩色实时输出（状态码/方法分色）
- ✅ 自动排除静态资源路径
- ✅ 按域名过滤查询
- ✅ 状态码和请求方法统计
- ✅ Django中间件一键集成
- ✅ 密码保护的Web可视化页面
- ✅ Chart.js图表统计（请求趋势、状态码分布）
- ✅ 纯Python编写，无外部依赖

## 环境变量

在项目根目录 `.env` 中配置：

```env
# 访问日志配置
ACCESS_LOG_ENABLED=True
ACCESS_LOG_PATH=logs/access.log
ACCESS_LOG_MAX_SIZE=10
ACCESS_LOG_BACKUP_COUNT=5
ACCESS_LOG_EXCLUDE_EXT=.ico,.css,.js,.png,.jpg,.jpeg,.gif,.svg,.woff,.woff2,.map
ACCESS_LOG_SHOW_CONSOLE=True
ACCESS_LOG_COLOR=True
ACCESS_LOG_PASSWORD=admin123
DEBUG=True
```

## 快速开始

### 1. Python 调用

```python
import sys
sys.path.insert(0, r"A:\MyProjects\WebModel\utils\模块文件数据库")
from access_logger import AccessLogger

# 初始化日志记录器
logger = AccessLogger()

# 记录一条请求
logger.log_request({
    'path': '/index',
    'method': 'GET',
    'ip': '127.0.0.1',
    'user_agent': 'Mozilla/5.0 Chrome/120.0',
    'status_code': 200,
    'duration': 0.123,
    'domain': 'example.com'
})

# 查询日志
logs, total = logger.get_logs(
    domain='example.com',
    limit=100,
    status_code=200
)
print(f"找到 {total} 条日志")

# 获取统计数据
stats = logger.get_statistics(domain='example.com', days=7)
print(f"总请求数: {stats['total_requests']}")
print(f"独立IP: {stats['unique_ip_count']}")
```

### 2. 命令行使用

### 3. Django 集成（推荐）

插件已自动配置好Django中间件和视图，只需确认配置。

**中间件已自动注册**（在 `WebModel/settings.py`）：
```python
MIDDLEWARE = [
    # ...
    'DZY_Web.middleware.access_log_middleware.AccessLogMiddleware',
    # ...
]
```

**路由已自动注册**（在 `DZY_Web/views/urls.py`）：
```
/web/{SITE_SLUG}/access-log/  → 访问日志可视化页面
```

**启动Django服务后，访问：**
```
http://你的域名/web/mysite/access-log/
```
默认密码：`admin123`（可在 `.env` 中修改 `ACCESS_LOG_PASSWORD`）

## 核心 API

### `AccessLogger` 类

| 方法 | 说明 | 参数 | 返回值 |
|------|------|------|--------|
| `__init__(...)` | 初始化日志记录器 | 可选：log_path, max_size, backup_count, exclude_ext, show_console, use_color, enabled, password | - |
| `log_request(data)` | 记录一条请求 | data: dict (path, method, ip, status_code 为必填) | bool (是否成功记录) |
| `get_logs(...)` | 查询日志 | 可选：domain, limit, offset, status_code, method, start_date, end_date | Tuple[List[Dict], int] (日志列表, 总数量) |
| `get_statistics(...)` | 获取统计数据 | domain: 可选, days: 统计最近N天 | Dict (包含各类统计指标) |
| `verify_password(password)` | 验证访问密码 | password: str | bool |

## 日志格式

每条日志以 JSON 格式存储，一行一条：

```json
{
  "timestamp": "2024-04-28 14:30:00",
  "domain": "example.com",
  "ip": "192.168.1.100",
  "method": "GET",
  "path": "/index",
  "status_code": 200,
  "duration": 0.045,
  "user_agent": "Mozilla/5.0 Chrome/120.0"
}
```

## 日志切割策略

- 当日志文件大小超过 `ACCESS_LOG_MAX_SIZE`（默认10MB）时自动滚动
- 旧日志自动 GZIP 压缩，命名格式：`access.log.1.gz`
- 最多保留 `ACCESS_LOG_BACKUP_COUNT`（默认5）个备份

## 终端彩色输出

终端实时输出采用分色显示：
- 🟢 **绿色**：2xx 成功状态码、GET方法
- 🔵 **蓝色**：3xx 重定向、POST方法
- 🟡 **黄色**：4xx 客户端错误、PUT方法
- 🔴 **红色**：5xx 服务器错误、DELETE方法

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 缺少必填字段 | 抛出 ValueError |
| 静态资源请求 | 自动跳过，不记录 |
| 日志写入失败 | 静默失败，不影响正常请求 |
| 密码错误 | 可视化页面提示错误 |
| 日志文件不存在 | get_logs 返回空列表 |

## 安全提示

1. **修改默认密码**：上线前务必修改 `.env` 中的 `ACCESS_LOG_PASSWORD`
2. **生产环境建议**：关闭终端彩色输出 `ACCESS_LOG_COLOR=False`
3. **排除敏感路径**：可在 `ACCESS_LOG_EXCLUDE_EXT` 中添加需要排除的路径后缀
4. **日志目录权限**：确保 logs 目录有写入权限
