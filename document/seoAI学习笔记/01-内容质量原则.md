# 内容质量原则 — Google 官方 SEO 指南

> 来源：Google Search Central 官方文档
> 核心页面：创建实用、可靠、以用户为中心的内容
> 学习日期：2026-04-27

---

## 一、核心原则：以用户为中心

Google 反复强调的**唯一核心**：内容必须主要是为用户而创建，而不是为了操纵搜索引擎排名。

### 有用内容的特征

| 特征 | 说明 |
|------|------|
| 受众明确 | 内容对网站的既有或目标受众直接有用 |
| 第一手经验 | 展示实际使用/到访/体验的经验和深度 |
| 定位清晰 | 网站有主要目的或关注点 |
| 满足需求 | 读者阅读后能充分了解主题、实现目标 |
| 体验良好 | 读者不需要再次搜索寻找更好信息 |

### 无用内容的警告信号

- 主要为吸引搜索流量而创作
- 大量自动化生成广泛主题
- 仅总结他人内容无额外价值
- 追逐流行话题而非服务现有读者
- 为字数而凑字数
- 为搜索流量进入毫无专业技能的领域
- 承诺回答没有答案的问题
- 未改内容仅改日期伪装新内容

## 二、E-E-A-T 原则

E-E-A-T = **经验 (Experience) + 专业性 (Expertise) + 权威性 (Authoritativeness) + 可信度 (Trustworthiness)**

### 关键要点

1. **可信度最重要**：其他方面影响可信度，但内容不一定要展示所有因素
2. **YMYL 主题要求极高**：可能影响健康、财务、安全的内容需要高度 E-E-A-T
3. **不是直接排名因素**：E-E-A-T 是评估内容质量的思维框架，识别优质 E-E-A-T 内容的综合因素会帮助排名

### "对象、方式、原因" 评估框架

| 维度 | 问题 | 行动 |
|------|------|------|
| **Who（对象）** | 内容的创建者是谁？ | 明确署名 + 展示作者背景和专业领域 |
| **How（方式）** | 内容如何创建？ | 透明展示制作过程；使用AI需披露 |
| **Why（原因）** | 为何创建？ | **最重要**。初衷必须让用户受益 |

### AI 内容的关键红线

> ⚠️ 如果使用 AI 制作内容的主要目的是**操纵搜索排名**，则违反 Google 网络垃圾政策。

正确做法：
- AI 辅助创作可以，但内容必须为用户服务
- 向读者披露 AI 的使用方式和原因
- AI 是工具，不是替代品

## 三、内容自检清单

### 内容与质量

- [ ] 是否提供原创信息、报告、研究或分析？
- [ ] 是否对主题提供了实质性、完整性描述？
- [ ] 是否提供富有洞察力的深刻见解？
- [ ] 若借鉴他人，是否提供了实质性额外价值？
- [ ] 标题是否描述性强且避免标题党？
- [ ] 内容是否值得被书签、分享或推荐？
- [ ] 质量是否达到可被出版物引用的标准？
- [ ] 与搜索结果中其他网页相比，是否有附加价值？
- [ ] 是否存在拼写或样式问题？
- [ ] 是否属于大规模外包生产？

### 专业知识

- [ ] 呈现方式是否值得信任（明确来源、作者简介）？
- [ ] 网站是否被公认为相关主题的权威？
- [ ] 是否由专家或爱好者撰写/审核？
- [ ] 是否存在可轻松验证的事实错误？

## 四、Google 明确指出"不需要重点关注"的误区

| 误区 | 真相 |
|------|------|
| 元关键字 (meta keywords) | Google 不使用关键字元标记 |
| 关键字堆砌 | 违反垃圾政策，且让用户厌烦 |
| 域名中的关键字 | 对排名几乎无影响 |
| 内容长度 | 没有最佳字数限制，长度本身不是排名因素 |
| 子网域 vs 子目录 | 对 SEO 无实质影响 |
| PageRank | 只是众多排名因素之一 |
| E-E-A-T 是排名因素 | 它不是直接排名因素，是思维框架 |

## 五、对 AI 写文章的启示

1. **选题阶段**：必须选择你有实际经验或专业知识的领域
2. **创作阶段**：AI 可以辅助，但必须注入第一手经验和独特见解
3. **署名阶段**：明确标注作者和背景，必要时披露 AI 辅助
4. **评估阶段**：每篇文章必须能通过上面的自检清单
5. **长期策略**：持续在特定领域输出，建立权威性
