# 网站结构与导航 — 内链、站点地图、规范化

> 来源：Google Search Central 官方文档
> 学习日期：2026-04-27

## 一、Google 搜索工作方式（三阶段）

理解搜索工作方式是做好 SEO 的基础：

### 阶段 1：抓取 (Crawling)

| 要点 | 说明 |
|------|------|
| 网址发现 | Google 通过链接和站点地图发现新网页 |
| 访问抓取 | Googlebot 下载文字、图片和视频 |
| 渲染 | 使用最新版 Chrome 渲染 JS |
| 抓取障碍 | 服务器问题、robots.txt 阻止、需登录 |

### 阶段 2：编入索引 (Indexing)

| 要点 | 说明 |
|------|------|
| 内容分析 | 分析文字、`<title>`、Alt 属性等 |
| 规范化聚类 | 将相似网页归组，选择规范网页 |
| 收集信号 | 语言、地区、可用性等 |
| 索引阻碍 | 质量低、noindex 标记、设计问题 |

### 阶段 3：呈现/排名 (Serving)

| 要点 | 说明 |
|------|------|
| 匹配 | 在索引中搜索匹配网页 |
| 排名 | 数百个因素决定相关性 |
| 个性化 | 位置、语言、设备影响结果 |
| **不收费** | Google 不通过收费提高排名 |

### 帮助 Google 发现你的网页

1. **提交站点地图** — 主动告知需要抓取的网址
2. **优化内部链接** — 从中心页（首页、分类页）链接到新内容
3. **使用关键标记** — 正确使用 `<title>` 和 Alt 属性
4. **确保 JS 可渲染** — Google 用 Chrome 渲染，确保内容可见

## 二、站点地图 (Sitemap)

### 作用

| 功能 | 说明 |
|------|------|
| 提供内容信息 | 网页、视频、文件的关系 |
| 提升抓取效率 | 搜索引擎更高效抓取 |
| 标识重要页面 | 告诉搜索引擎哪些页面重要 |
| 提供元数据 | 更新时间、备用语言版本 |
| 富媒体内容 | 视频时长/评分、图片位置、新闻标题 |

### 需要站点地图的场景

| 需要 ✅ | 不需要 ❌ |
|---------|----------|
| 网站很大（>500页） | 小型网站（<500页） |
| 新站且外链少 | 内部全面建立链接 |
| 大量富媒体内容 | 不需要视频/图片/新闻搜索展示 |

> 站点地图不保证 Google 能抓取和索引所有内容，但大多数网站都会受益。

## 三、URL 规范化 (Canonical)

### 为什么需要规范化

- 同一内容可能通过多个 URL 访问（如 `example.com` 和 `www.example.com`）
- Google 会自动将相似网页聚类，选择"规范网页"
- 自动选择可能浪费抓取资源或选错

### 最佳实践

1. **每项内容只通过一个 URL 访问** — 最根本的解决方案
2. **使用 `rel="canonical"`** — 明确告诉 Google 规范网址
3. **重定向** — 将非首选 URL 301 重定向到规范 URL

## 四、内部链接策略

### 核心原则

1. **每个重要页面至少被 1 个其他页面链接**
2. **在相关上下文中提供内链** — 帮助用户和 Google 理解关系
3. **使用描述性锚文本** — "有道翻译批量处理脚本" 优于 "点击这里"
4. **新内容要链接到旧内容** — 从相关文章交叉引荐
5. **旧内容要链接到新内容** — 在相关旧文中添加新文章链接

### 网站目录组织

- 将主题相似的网页归入同一目录（如 `/news/`、`/faq/`）
- 帮助 Google 了解不同目录的更新频率
- 目录结构影响 Google 的抓取优先级

## 五、robots.txt 与控制抓取

### robots.txt

- 告诉 Googlebot 哪些页面可以/不可以抓取
- **注意**：阻止抓取 ≠ 阻止索引（Google 可能从其他来源获取信息并索引）
- 要阻止索引，应使用 `noindex` 元标记

### noindex

```html
<meta name="robots" content="noindex">
```

- 明确告诉搜索引擎**不要索引此页面**
- 比 robots.txt 更可靠地阻止页面出现在搜索结果中

## 六、对文章网站的具体建议

### 当前网站结构

```
/web/youdaotools/          → 首页（最新6篇）
/web/youdaotools/news/     → 新闻列表（分页）
/web/youdaotools/news/<id>/ → 文章详情
/web/youdaotools/faq/      → FAQ
```

### 改进建议

1. **添加站点地图** — 当前网站没有 sitemap.xml
2. **文章间交叉链接** — 在详情页的"相关文章"基础上，文章正文内也应互链
3. **面包屑导航** — 帮助 Google 理解网站层级
4. **规范 URL** — 确保同一文章只有一个可访问 URL
5. **404 页面优化** — 帮助用户回到有效页面
