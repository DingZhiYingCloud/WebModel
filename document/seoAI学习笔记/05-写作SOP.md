# SEO 写作 SOP — 从选题到发布的完整流程

> 本文件定义了写一篇 SEO 优化文章的具体步骤。
> 配合 `00-快速索引.md` 使用，每一步都需通过对应检查项。

---

## SOP 总览

```
选题 → 研究 → 大纲 → 写作 → SEO优化 → 发布前检查 → 发布 → 发布后
```

---

## Step 1：选题与定位

### 1.1 确定选题

| 行动 | 说明 |
|------|------|
| 选择你**有经验**的领域 | E-E-A-T 中的 Experience 是第一优先级 |
| 检查搜索意图 | 用户搜这个关键词想得到什么？ |
| 确认选题价值 | 能否提供独特见解或实际帮助？ |
| 避免追逐热点 | 如果不是你的专业领域，不要为了流量硬写 |

### 1.2 关键词研究

| 行动 | 说明 |
|------|------|
| 确定核心关键词 | 用户最可能搜索的词 |
| 找出相关变体 | 同义词、相关词、长尾词 |
| 竞品分析 | 搜索核心关键词，看排名前5的内容有什么，你能提供什么额外价值 |

### 1.3 输出

- 核心关键词 + 3-5个变体
- slug（URL路径名）
- 一句话描述目标读者和文章价值

---

## Step 2：内容研究

### 2.1 收集素材

| 行动 | 说明 |
|------|------|
| 第一手经验 | 你自己是否使用过/体验过？记录下来 |
| **浏览器研究** | **原创文章必须用浏览器实际访问官网/操作功能/截图记录** |
| 权威来源 | 引用 Google 官方文档、学术论文、行业报告 |
| 用户痛点 | 论坛、社区、评论中用户的真实问题 |
| 竞品缺口 | 排名靠前的文章缺了什么？ |

### 2.2 原创文章的研究流程（article-creator skill 使用）

```
1. 确定研究方向 → 产品/工具类必须访问官网
2. 浏览器实际体验 → 操作功能、截图记录
3. 查阅官方文档 → API参考、技术细节
4. 搜索补充资料 → 用户讨论、行业数据
5. 整理研究笔记 → 结构化记录观察和见解
```

**研究笔记模板**：
```
- 主题：xxx
- 官网观察：xxx（含截图记录）
- 核心功能/要点：xxx
- 用户痛点/常见问题：xxx
- 独特见解（基于实际体验）：xxx
- 可用图片清单：xxx
```

### 2.3 注意事项

- 不要仅总结他人内容（Google 认为这是低价值）
- 必须有**实质性额外价值**
- 引用来源时添加外链
- **原创文章严禁凭空编造**，所有内容必须有研究支撑

---

## Step 3：构建大纲

### 3.1 标题层级

```
H1: 文章标题（含核心关键词）
  H2: 主要章节
    H3: 子章节
  H2: 另一个主要章节
    H3: 子章节
  H2: 总结/行动指南
```

### 3.2 大纲必须包含

| 要素 | 说明 |
|------|------|
| 引言 | 痛点切入 + 文章价值预告 |
| 核心内容 | 按逻辑递进，每节一个小主题 |
| 实操部分 | 步骤、代码、截图、示例 |
| 常见问题 | 回答用户可能的疑问 |
| 总结 | 关键要点 + 行动建议 |

### 3.3 内链规划

在大纲阶段就规划好：
- 哪些旧文章需要链接到本文
- 本文需要链接到哪些旧文章
- 自然融入，不要硬塞

---

## Step 4：撰写内容

### 4.1 写作原则

| 原则 | 说明 |
|------|------|
| 开门见山 | 第一段就回答用户的核心问题 |
| 段落简短 | 每段 3-5 句话，便于阅读 |
| 小标题分隔 | 每 300-500 字一个小标题 |
| 列表优先 | 步骤、要点用有序/无序列表 |
| 代码可复制 | 技术文章的代码必须完整可运行 |
| 图片辅助 | 关键步骤配截图或示意图 |

### 4.2 关键词使用

| 位置 | 要求 |
|------|------|
| `<title>` | 核心关键词前置 |
| `<h1>` | 与 title 一致或相似 |
| 前100字 | 自然包含核心关键词 |
| 小标题 | 适当包含变体词 |
| 正文 | 自然融入，不堆砌 |
| 图片 alt | 描述性文本，含关键词 |

### 4.3 AI 辅助写作的规则

| 可以 | 不可以 |
|------|--------|
| AI 辅助组织结构 | AI 生成后不审核直接发布 |
| AI 辅助润色语言 | 完全依赖 AI 无第一手经验 |
| AI 辅助生成初稿 | 隐瞒 AI 使用（如果主要内容由 AI 生成） |

## Step 5：SEO 优化

### 5.1 页面元素检查

按 `00-快速索引.md` 中的清单逐项检查：

```
□ <title> — 独立、简洁、含关键词、≤60字符
□ meta description — 独特、概括、含关键信息、≤160字符
□ slug — 简短英文、连字符分隔
□ <h1> — 清晰、醒目、1个
□ 图片 alt — 描述性、含关键词
□ 结构化数据 — Article JSON-LD
□ 内链 — 2-3个相关文章链接
□ 外链 — 引用来源有链接
```

### 5.2 结构化数据填写

必填项：
- `headline` — 文章标题
- `image` — 至少1张高分辨率图片（推荐16:9, 4:3, 1:1三种）
- `datePublished` — 发布时间（ISO 8601）
- `dateModified` — 修改时间（ISO 8601）
- `author.name` — 作者名（仅名字）
- `author.url` — 作者页面链接

---

## Step 6：发布前最终检查

### 6.1 内容质量检查

```
□ 读者看完后是否不需要再搜索？
□ 是否有独特见解或第一手经验？
□ 事实是否准确可验证？
□ 是否有拼写/语法错误？
□ 是否值得被书签/分享？
```

### 6.2 技术 SEO 检查

```
□ 页面可正常访问
□ 图片可正常显示
□ 内链指向正确
□ 结构化数据通过富媒体搜索结果测试
□ 移动端显示正常
```

### 6.3 发布操作

1. 通过 API 创建文章（title, slug, html_content, cover_image）
2. 确认文章在网站上可正常访问
3. 从相关旧文章添加内链到新文章

---

## Step 7：发布后维护

### 7.1 短期（发布后1周内）

- 检查页面是否被 Google 收录（`site:` 搜索）
- 确认搜索结果中 title 和 description 显示正确
- 监控初始流量

### 7.2 长期（每月/每季度）

- 检查文章内容是否需要更新
- 更新过时信息（修改 `dateModified`）
- 添加新的内链
- 优化表现不佳的文章
- 删除确实无法挽回的内容

### 7.3 应对排名下降

1. 检查是否受核心更新影响
2. 客观评估内容质量
3. 有意义地改进内容（非表面修补）
4. 耐心等待（几天到几个月）
