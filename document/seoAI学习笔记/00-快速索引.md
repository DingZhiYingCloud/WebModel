# SEO 快速索引 — AI 写文章的 5 分钟速查表

> 本文件是其他 AI 快速掌握 SEO 写作规范的核心参考。
> 详细内容见同目录下的 01-04 号文件。

---

## 一、写文章前必须记住的 4 条铁律

1. **为用户而写，不为搜索引擎而写** — 内容必须对读者有实际帮助
2. **有第一手经验 > 纯搬运** — Google 重视 Experience（经验）
3. **AI 辅助可以，但不能只为了排名而用 AI** — 透明披露 AI 使用
4. **先研究再写作** — 原创文章必须先实际体验/研究主题（访问官网、操作功能、查阅文档），不能凭空编造

---

## 二、文章 SEO 自检清单（每篇必过）

### 内容层

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 原创性 | 有独特见解/经验/分析，非单纯改写他人内容 |
| 2 | 完整性 | 读者看完后不需要再搜索其他来源 |
| 3 | 准确性 | 事实可验证，无错误 |
| 4 | 可读性 | 分段清晰，标题层级正确，无拼写错误 |
| 5 | 时效性 | 信息是最新的，或明确标注时效范围 |

### 页面层

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 6 | `<title>` | 每页独立、简洁描述内容、含核心关键词、≤60字符 |
| 7 | meta description | 每页独特、概括全页、含关键信息、≤160字符 |
| 8 | URL (slug) | 简短描述性、用连字符分隔、小写英文 |
| 9 | `<h1>` | 每页1个、与 title 一致或相似、清晰醒目 |
| 10 | 图片 | 有 alt 文本、放在相关文字附近、高画质 |

### 结构层

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 11 | 内链 | 至少 2-3 个相关文章的内部链接 |
| 12 | 外链 | 引用权威来源时添加外链 |
| 13 | 结构化数据 | Article JSON-LD（headline/author/datePublished/dateModified/image） |

## 三、Title / Description / URL 写法速查

### Title 公式

```
[核心关键词] - [补充说明] | [网站名]
```

示例：
- ✅ `有道翻译批量处理脚本指南 - Python自动化翻译 | 有道翻译`
- ❌ `有道翻译 批量翻译 翻译脚本 翻译工具 翻译API`
- ❌ `文章标题`

### Meta Description 公式

```
[核心价值主张]。[关键信息1]，[关键信息2]。
```

示例：
- ✅ `完整的Python批量翻译脚本教程，支持有道翻译API，实现文件夹自动翻译，含编码处理和错误排查指南。`
- ❌ `翻译工具，批量翻译，脚本翻译，自动翻译`
- ❌ `本文介绍了...` （太空泛）

### URL (slug) 公式

```
[简短英文关键词]-[补充说明]
```

示例：
- ✅ `youdao-batch-translation-script-guide`
- ❌ `article-123`
- ❌ `有道翻译批量处理脚本指南`

## 四、结构化数据模板（Article）

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "NewsArticle",
  "headline": "文章标题",
  "image": [
    "https://example.com/photos/16x9/photo.jpg",
    "https://example.com/photos/4x3/photo.jpg",
    "https://example.com/photos/1x1/photo.jpg"
  ],
  "datePublished": "2026-04-27T08:00:00+08:00",
  "dateModified": "2026-04-27T09:20:00+08:00",
  "author": [{
    "@type": "Person",
    "name": "作者名",
    "url": "https://example.com/profile/author"
  }]
}
</script>
```

## 五、关键词使用原则

| 原则 | 说明 |
|------|------|
| 自然融入 | 不要生硬塞入，读起来要通顺 |
| 覆盖变体 | 同一概念的不同说法（如"翻译工具"/"翻译软件"/"翻译器"） |
| 标题必含 | `<title>` 和 `<h1>` 必须包含核心关键词 |
| 不堆砌 | 同一关键词不要在同一位置重复超过2次 |
| 长尾优先 | "有道翻译批量处理Python脚本" 比 "翻译工具" 更精准 |

## 六、Google 明确不关注的因素

以下内容**不要浪费时间优化**：

- ❌ meta keywords（Google 不使用）
- ❌ 内容长度（无最佳字数，够用就行）
- ❌ 域名中的关键词
- ❌ 子网域 vs 子目录
- ❌ 标题标签的数量和顺序
