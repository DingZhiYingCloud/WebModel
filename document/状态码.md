# WebGPT 全局状态码规范

## 状态码分类说明

| 分类 | code 范围 | 说明 |
|------|-----------|------|
| 成功 | 0 | 请求处理成功 |
| 客户端错误 | 400 - 499 | 参数缺失、格式错误等 |
| 服务端错误 | 500 - 599 | 服务器内部异常 |
| 业务自定义 | 1000+ | 各模块业务逻辑错误 |

---

## 通用状态码（HTTP 层）

| 状态码 | 名称 | 说明 |
|--------|------|------|
| **0** | SUCCESS | 请求成功 |
| **400** | BAD_REQUEST | 请求参数错误 |
| **401** | UNAUTHORIZED | 未授权/未登录 |
| **403** | FORBIDDEN | 无权限访问 |
| **404** | NOT_FOUND | 资源不存在 |
| **405** | METHOD_NOT_ALLOWED | 请求方式不允许 |
| **500** | INTERNAL_SERVER_ERROR | 服务器内部错误 |

---

## 业务通用状态码

| 状态码 | 名称 | 说明 |
|--------|------|------|
| **1001** | PARAM_MISSING | 必填参数缺失 |
| **1002** | PARAM_INVALID | 参数格式/值不合法 |
| **1003** | PARAM_TOO_LONG | 参数长度超限 |
| **1004** | DATA_NOT_FOUND | 数据不存在 |

---

## 使用规则

1. 所有 API 统一使用 `JsonResponse({"code": xxx, "message": "描述", "data": ...})` 格式
2. 错误场景 `data` 字段统一传 `None`
3. 成功场景 `data` 字段传递实际数据或 `{}`（无数据时）
4. 各模块可在 **1000+** 范围内定义专属状态码，避免冲突
